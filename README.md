# MMWave Radar AI Project

This repository contains the complete implementation of an MMWave Radar AI system for metal object detection.

## üéØ Project Overview

This project demonstrates radar signal processing and AI-based classification using synthetic FMCW radar data. It achieves **99% classification accuracy** on metal vs. non-metal detection with CNN, and **96% with SVM** in comparative analysis.

## üìÅ Project Structure

### **Part 1: Synthetic Radar Signal Processing** ‚úÖ
-   `radar_simulation.ipynb`: Generates synthetic FMCW radar signals and visualizes Range-Doppler heatmaps for Empty, Metal, and Clutter scenarios.

### **Part 2: Metal vs Non-Metal Classification** ‚úÖ
-   `classification_model.ipynb`: Trains a CNN to classify Metal vs Non-Metal with **99.5% test accuracy**.
-   `model_comparison.ipynb`: **Comparative analysis** of CNN vs SVM (CNN: 99.5%, SVM: 96%).
-   `radar_cnn_model.pth`: Trained model weights (generated by notebook).
-   `radar_utils.py`: Shared utility library for radar signal generation and processing.

### **Part 3: Hidden Object Detection Challenge** ‚úÖ
-   `hidden_object_detection.ipynb`: Demonstrates background subtraction technique to detect hidden metal objects in clutter.

### **Part 4: Deployment Logic** ‚úÖ
-   `deployment_design.pdf`: Complete design document for real-time deployment.

### **Part 5: Demo Video** ‚úÖ
-   `demo_video`: Recorded video explaining approach, synthetic data generation, CNN model, etc.

## üìä Key Results

### Classification Performance
- **CNN Accuracy**: 99.5% (199/200 correct)
  - Precision: 99% (Non-Metal), 100% (Metal)
  - Recall: 100% (Non-Metal), 99% (Metal)
  - F1-Score: 1.00 (Non-Metal), 0.99 (Metal)

- **SVM Accuracy**: 96.0% (192/200 correct)
  - Precision: 95% (Non-Metal), 98% (Metal)
  - Recall: 98% (Non-Metal), 94% (Metal)
  - F1-Score: 0.96 (both classes)

### Model Comparison
| Metric | CNN | SVM |
|--------|-----|-----|
| **Test Accuracy** | 99.5% | 96.0% |
| **Training Time** | 223.69s | 34.28s |
| **Model Complexity** | High | Medium |
| **Feature Engineering** | Automatic | Manual |

### Why CNN Outperforms SVM:
1. **Automatic Feature Learning**: CNN learns spatial features from Range-Doppler maps
2. **Better Generalization**: Handles unseen scenarios more effectively
3. **Scalability**: Can be extended to multi-class classification and object tracking
4. **Edge Deployment**: Optimizable with INT8 quantization for real-time inference

## üì¶ Core Components

### `radar_utils.py`
Shared utility library containing:
- `generate_radar_cube()`: FMCW radar signal simulation
- `process_radar_cube()`: Range-Doppler processing (2D FFT)
- `get_random_scenario()`: Scenario generation (metal, non-metal, clutter, hidden)

### Configuration
- **Frequency**: 77 GHz (automotive radar)
- **Bandwidth**: 4 GHz
- **Chirps**: 128 per frame
- **Samples**: 600 per chirp
- **Dataset**: 1000 synthetic samples (balanced)

## üîß Technical Highlights

1. **Realistic FMCW Radar Simulation**: Physics-based signal generation
2. **Global Normalization Fix**: Preserves amplitude differences between classes (key innovation)
3. **Comparative Analysis**: Evaluated CNN vs SVM to justify model selection
4. **Background Subtraction**: Detects hidden objects in clutter
5. **Edge Deployment Ready**: INT8 quantization for low-latency inference (<50ms)
